# Prompt Launcher v1 (macOS) Implementation Plan

> **For Hermes:** Use `subagent-driven-development` skill to implement this plan task-by-task. Each task should be dispatched as a fresh `delegate_task` with the task's section pasted verbatim into the goal, plus the relevant context block from below.
>
> **Authoritative sources:** dossier `~/.hermes/ideas/research/007-prompt-library-launcher.md` (§ 4 tech sketch, § 8 spike verdict), spike README `~/spikes/prompt-launcher-core/001-hotkey-paste-loop/README.md`, backlog `~/.hermes/ideas/backlog.md` (#007 YAML — `stack_choice`, `macos_integration_shape`, all `known_gotcha:` fields).

**Goal:** A macOS-only Tauri 2 app, code-signed `.app` bundle, that binds a global hotkey (default `Cmd+Shift+Space`) to a Spotlight-style overlay; users fuzzy-search prompts from `~/.prompts/*.md` and Enter pastes the selected prompt at the cursor in the previously-focused app.

**Architecture:** The spike (`~/spikes/prompt-launcher-core/001-hotkey-paste-loop/`) validated the load-bearing flow at ~170ms hotkey→paste. v1 keeps that exact flow and adds three commodity layers around it: a `~/.prompts/` markdown loader with hot-reload, a `nucleo`-backed fuzzy-search UI in a webview, and last-used ordering. Everything else (variables, per-app routing, Linux, Wayland) is explicitly deferred to v2 — the v2 surface is sketched in § "Forward-compat seams" so we don't paint ourselves into a corner.

**Tech Stack:**
- Rust 2024, MSRV 1.83 (match the spike)
- `tauri` 2.11 (`features = ["macos-private-api"]`)
- `tauri-plugin-global-shortcut` 2.x
- `arboard` 3 (clipboard)
- `enigo` 0.2 (synthetic Cmd+V via `Key::Other(9)`)
- `objc2-app-kit` 0.2 (`features = ["NSWorkspace", "NSRunningApplication", "NSApplication", "libc"]`)
- `nucleo` 0.5 (Helix/Zed's fuzzy matcher — **not** `nucleo-matcher` directly; we want the higher-level `Nucleo` API)
- `notify` 6 (filesystem watching)
- `serde` 1 + `serde_yaml` 0.9 (frontmatter)
- `gray_matter` 0.2 (markdown frontmatter splitter — saves us rolling our own)
- `directories` 5 (find `~/.prompts/` cross-platform-ish, though we only target macOS)
- Webview UI: **vanilla TypeScript + Vite + Tauri's `@tauri-apps/api`**, no UI framework — the overlay has ~3 interactive elements (input, list, status row). Adding Svelte/Solid/React buys nothing here and adds 100ms to cold-start.

**Build path:** standalone-rust-workspace at `~/code/prompt-launcher/`. **Not** Hermes-integrated — this is an end-user desktop app, not an agent integration.

**v1 scope (deliberately small — from dossier § 4, refined by spike):**
1. Global hotkey toggle (default `Cmd+Shift+Space`, hard-coded in v1; configurable in v1.x).
2. Overlay = frameless, transparent, always-on-top, `~600×400` webview centered on active display; `ActivationPolicy::Accessory`.
3. Prompt store = `~/.prompts/**.md` (recurse arbitrarily deep, one tag-per-subdirectory convention but no enforcement). Frontmatter: `title` (string, required), `tags` (list of strings, optional), `last_used` (ISO timestamp, optional — managed by app, not user).
4. Fuzzy search via `nucleo`. Matches against `title + tags-joined-by-space + body`. Title matches weighted highest.
5. Selection → clipboard-stash-then-paste flow (the exact loop the spike validated).
6. Last-used ordering: bump `last_used` in the frontmatter of the selected file on Enter; sort merges nucleo score + recency.
7. First-run onboarding: detect Accessibility permission, open `x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility` if missing, block usage until granted.
8. CGEvent-based timing harness — a `#[cfg(test)]` integration test in `app-core` that asserts hotkey→paste completes in `<200ms` against a controlled-input scenario.

**v2 deferred (acknowledged, not built):** Linux/X11, Wayland (`libei`/`ydotool` story), variables (`{{filename}}`, `{{selection}}`), per-app routing, multi-step prompts, sync, Windows. The user named **kicking off agent workflows (e.g. Smithers)** as a v2 direction — captured in `## Forward-compat seams` below so v1's prompt schema and selection flow can be extended without breaking changes.

## Forward-compat seams (v2 will need these; v1 must not block them)

Even though v2 is deferred, three forward-compat hooks earn their keep in v1:

1. **Prompt action trait, not hardcoded paste.** `crates/app-core` defines `trait PromptAction { fn execute(&self, ctx: &Context) -> Result<()> }` with one implementation in v1 (`PasteAction`). v2 will add `AgentAction { command: String, args: Vec<String> }` for the Smithers-style "fire an agent workflow" case. The trait + a `serde`-tagged `kind: paste` field in the prompt frontmatter is the entire seam.
2. **`Context` struct, not naked function args.** `execute()` receives `&Context { prompt, focused_app_pid, focused_window_title, selection_text: Option<String> }`. v1 only fills `prompt + focused_app_pid` but the field set is locked. Per-app routing (v2) is a single `Context.focused_window_title`-based filter on the search results; variables (v2) read from `Context.selection_text`.
3. **Storage backend trait.** `trait PromptStore { fn list(&self) -> Vec<Prompt>; fn touch(&self, id: &str) -> Result<()>; fn watch(&self, sink: Sender<Event>); }`. v1 has one impl (`FsPromptStore` reading `~/.prompts/`). v2 sync (git-tracked vault, or dedicated service) is a second impl. No premature features — just the trait shape.

**No other v1 work is gated on forward-compat.** Specifically: the `kind: paste` frontmatter field has a default so existing markdown prompts (with no `kind:` field) load as `PasteAction`; users don't need to migrate.

```rust
use objc2_app_kit::{NSWorkspace, NSRunningApplication};

/// Apple's NSApplicationActivateIgnoringOtherApps = 1 << 1 (= 2).
/// objc2-app-kit 0.2 doesn't expose this as an enum variant; we define it inline.
const ACTIVATE_IGNORING_OTHER_APPS: u64 = 1 << 1;

pub fn capture_frontmost_pid() -> Option<i32> {
    unsafe {
        let workspace = NSWorkspace::sharedWorkspace();
        let app = workspace.frontmostApplication()?;
        Some(app.processIdentifier())
    }
}

pub fn restore_focus(pid: i32) -> bool {
    unsafe {
        let Some(app) = NSRunningApplication::runningApplicationWithProcessIdentifier(pid) else {
            tracing::warn!(pid, "no running app for pid");
            return false;
        };
        app.activateWithOptions(ACTIVATE_IGNORING_OTHER_APPS.into())
    }
}
```

**Step 3:** `src/paste.rs` — copy the spike's paste flow:

```rust
use std::thread;
use std::time::Duration;
use arboard::Clipboard;
use enigo::{Enigo, Key, Direction, Keyboard, Settings};

/// kVK_ANSI_V virtual keycode for the V key on macOS.
/// enigo 0.2 Key::Unicode('v') traps with SIGTRAP when Cmd is held.
const KVK_ANSI_V: u16 = 9;

pub fn paste_text(text: &str) -> anyhow::Result<()> {
    let mut clipboard = Clipboard::new()?;
    let prior = clipboard.get_text().ok();

    clipboard.set_text(text)?;
    tracing::debug!("clipboard set");

    // Synthesize Cmd+V. Wait for focus to settle first (150ms — spike-validated).
    thread::sleep(Duration::from_millis(150));
    let mut enigo = Enigo::new(&Settings::default())?;
    enigo.key(Key::Meta, Direction::Press)?;
    enigo.key(Key::Other(KVK_ANSI_V as u32), Direction::Click)?;
    enigo.key(Key::Meta, Direction::Release)?;
    tracing::debug!("cmd+v synthesized");

    // Restore prior clipboard after paste settles (200ms — spike).
    if let Some(prior) = prior {
        thread::sleep(Duration::from_millis(200));
        clipboard.set_text(prior)?;
        tracing::debug!("clipboard restored");
    }
    Ok(())
}
```

**Step 1:** Extend `run()`:

```rust
use tauri::{ActivationPolicy, Manager};
use tauri_plugin_global_shortcut::{Code, GlobalShortcutExt, Modifiers, Shortcut, ShortcutState};

const HOTKEY: Shortcut = Shortcut::new(Some(Modifiers::SUPER.union(Modifiers::SHIFT)), Code::Space);

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .plugin(
            tauri_plugin_global_shortcut::Builder::new()
                .with_shortcut(HOTKEY).unwrap()
                .with_handler(|app, shortcut, event| {
                    if shortcut == &HOTKEY && event.state == ShortcutState::Pressed {
                        if let Some(win) = app.get_webview_window("overlay") {
                            let _ = win.show();
                            let _ = win.set_focus();
                        }
                    }
                })
                .build(),
        )
        .setup(|app| {
            #[cfg(target_os = "macos")]
            app.set_activation_policy(ActivationPolicy::Accessory);
            Ok(())
```

## Decision questions answered up-front

- **Why no UI framework?** Cold-start matters; overlay must feel instant. Vanilla TS + Vite is ~5KB JS for our needs vs. ~30KB+ for Svelte/Solid/React.
- **Why `nucleo` 0.5 over `nucleo-matcher`?** `Nucleo` is the higher-level streaming API; `nucleo-matcher` is the underlying matcher. v1 wants the streaming API because the UI feeds chars one at a time.
- **Why `gray_matter` instead of rolling our own splitter?** 90 lines saved; one less thing to test; battle-tested on tens of thousands of static-site-generator builds.
- **Why `chrono` not `time`?** App-core uses `DateTime<Utc>` in `Frontmatter`; `chrono` has the ergonomic API for what we need. `time` is technically better-engineered but the API ergonomics don't pay back here.
- **Why ad-hoc codesign instead of a real Developer ID?** v1 is personal-use; ad-hoc satisfies TCC's bundle-required check. Real signing + notarization is v1.x when we hand the app to anyone else.
- **Why hard-coded hotkey?** v1.x adds settings. Configurable hotkey requires a settings window + hotkey-conflict-detection UI that's not on the critical path.
